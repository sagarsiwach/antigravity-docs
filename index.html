Claude Pro Setup Guide

Access Claude Opus 4.5 with Extended Thinking via Claude Code CLI

What you're setting up: A separate Claude Code configuration that connects to our proxy server. This gives you access to Claude Opus 4.5 with extended thinking, Claude Sonnet 4.5, and Gemini models - while keeping your existing Claude Code setup completely intact.

Prerequisites

Before you begin, make sure you have:

  • Claude Code CLI installed - If not, install it first: npm install -g @anthropic-ai/claude-code
  • Your API key - You should have received this from your provider (starts with trial_)
  • Terminal access - Terminal.app on Mac, PowerShell on Windows

Step 1: Enter Your API Key

Paste your API key below. The instructions will automatically update with your key included, so you can copy and paste directly.

Step 2: Follow Setup Instructions

How this works: We're creating a separate config folder (~/.claude-pro) with its own settings. Then we create an alias claude-pro that uses this config. Your regular claude command stays unchanged.
Step 1 of 3
Create config folder and settings file

This command creates a new folder ~/.claude-pro in your home directory and writes the configuration file with your API key and proxy settings. The settings tell Claude Code which server to connect to and which AI models to use.

mkdir -p ~/.claude-pro && cat > ~/.claude-pro/settings.json << 'EOF'
{
  "showWelcomeTutorial": false,
  "hasCompletedOnboarding": true,
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "YOUR_API_KEY",
    "ANTHROPIC_BASE_URL": "https://proxy.classicgroup.asia",
    "ANTHROPIC_MODEL": "claude-opus-4-5-thinking",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-opus-4-5-thinking",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-5-thinking",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "gemini-3-flash",
    "CLAUDE_CODE_SUBAGENT_MODEL": "claude-sonnet-4-5-thinking"
  }
}
EOF
Step 2 of 3
Add the claude-pro alias to your shell

This adds a shortcut command called claude-pro to your shell configuration. The alias tells Claude Code to use the config folder we just created instead of the default one. The source command reloads your shell config so the alias works immediately.

echo "alias claude-pro='CLAUDE_CONFIG_DIR=~/.claude-pro claude'" >> ~/.zshrc && source ~/.zshrc

Using Bash instead of Zsh? Replace ~/.zshrc with ~/.bashrc in the command above. Most modern Macs use Zsh by default.

Step 3 of 3
Launch Claude Pro

That's it! Run this command to start Claude Code with your new configuration. You should see Claude start up and connect to the proxy server.

claude-pro
You're done! From now on, use claude-pro to access Claude Opus 4.5 with extended thinking. Your regular claude command (if you have one) remains unchanged.
How this works: We're creating a separate config folder in your user profile with its own settings. Then we add a PowerShell function claude-pro that uses this config. Your regular claude command stays unchanged.
Step 1 of 4
Open PowerShell as Administrator

Right-click the Start button and select "Windows Terminal (Admin)" or "PowerShell (Admin)". Running as administrator ensures you have permission to create the profile file if needed.

Step 2 of 4
Create the config folder

This creates a new folder called .claude-pro in your user profile directory (typically C:\Users\YourName). The -Force flag means it won't error if the folder already exists.

New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude-pro"
Step 3 of 4
Create the settings file

This creates the configuration file with your API key and proxy settings. The @'...'@ syntax is a PowerShell "here-string" that lets us write multi-line text. Copy and paste this entire block as one command.

@'
{
  "showWelcomeTutorial": false,
  "hasCompletedOnboarding": true,
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "YOUR_API_KEY",
    "ANTHROPIC_BASE_URL": "https://proxy.classicgroup.asia",
    "ANTHROPIC_MODEL": "claude-opus-4-5-thinking",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-opus-4-5-thinking",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-5-thinking",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "gemini-3-flash",
    "CLAUDE_CODE_SUBAGENT_MODEL": "claude-sonnet-4-5-thinking"
  }
}
'@ | Out-File -FilePath "$env:USERPROFILE\.claude-pro\settings.json" -Encoding UTF8
Step 4 of 4
Add the claude-pro function to your PowerShell profile

This adds a function called claude-pro to your PowerShell profile, which runs automatically when you open PowerShell. The first line creates the profile file if it doesn't exist. The second line adds the function. The third line reloads your profile so the function works immediately.

if (!(Test-Path $PROFILE)) { New-Item -ItemType File -Force -Path $PROFILE }
Add-Content $PROFILE "`nfunction claude-pro { `$env:CLAUDE_CONFIG_DIR = `"$env:USERPROFILE\.claude-pro`"; claude }"
. $PROFILE
Done!
Launch Claude Pro

Run this command to start Claude Code with your new configuration. You should see Claude start up and connect to the proxy server.

claude-pro
You're done! From now on, use claude-pro in PowerShell to access Claude Opus 4.5 with extended thinking. Your regular claude command (if you have one) remains unchanged.

Available Models

Troubleshooting

"Quota exhausted" or rate limit errors
What's happening: The AI model has a limit on how many requests can be made per minute/hour. When you hit this limit, you'll see this error temporarily.
What to do: Simply retry your request. The system automatically rotates between multiple accounts, so retrying usually works immediately. If it persists, wait 30-60 seconds and try again.
Connection timeout or "could not connect"
What's happening: Your computer couldn't reach the proxy server. This is usually a network issue on your end.
What to do: Check your internet connection. Try opening https://proxy.classicgroup.asia/health in your browser - you should see a response. If not, the server may be temporarily down; try again in a few minutes.
"Model not found" error
What's happening: You tried to use a model name that doesn't exist or is misspelled.
What to do: Use the exact model names listed above. They're case-sensitive - claude-opus-4-5-thinking works, but Claude-Opus-4-5-Thinking won't.
"claude-pro: command not found"
What's happening: The alias/function wasn't added to your shell config, or your shell hasn't reloaded the config.
What to do: Close your terminal completely and open a new one. If that doesn't work, re-run Step 2 (macOS) or Step 4 (Windows) from the instructions above.
Settings not taking effect
What's happening: Claude Code might be reading from a different config file, or the settings file has a syntax error.
What to do: Make sure your API key doesn't have extra spaces or quotes around it. You can verify the config file exists by running: cat ~/.claude-pro/settings.json (Mac/Linux) or type $env:USERPROFILE\.claude-pro\settings.json (Windows).

Need help? Contact your provider with the exact error message you're seeing.
