Claude Pro Setup Guide

Access Claude Opus 4.5 with Extended Thinking via Claude Code CLI

What you're setting up: A command alias that connects Claude Code to our proxy server. This gives you access to Claude Opus 4.5 with extended thinking, Claude Sonnet 4.5, and Gemini models.

Prerequisites

Before you begin, make sure you have:

  • Claude Code CLI installed - If not, install it first: npm install -g @anthropic-ai/claude-code
  • Your API key - You should have received this from your provider
  • Terminal access - Terminal.app on Mac, PowerShell on Windows

Step 1: Enter Your API Key

Paste your API key below. The instructions will automatically update with your key included.

Step 2: Follow Setup Instructions

How this works: We create an alias claude-pro that sets the API key and proxy URL as environment variables, then launches Claude Code. Your regular claude command stays unchanged.
Step 1 of 2
Add the claude-pro alias to your shell

This adds a shortcut command called claude-pro to your shell configuration. The alias sets your API key and proxy URL, then launches Claude Code.

echo "alias claude-pro='ANTHROPIC_AUTH_TOKEN=YOUR_API_KEY ANTHROPIC_BASE_URL=https://proxy.classicgroup.asia claude'" >> ~/.zshrc && source ~/.zshrc

Using Bash instead of Zsh? Replace ~/.zshrc with ~/.bashrc in the command above. Most modern Macs use Zsh by default.

Step 2 of 3
Launch Claude Pro

Run this command to start Claude Code.

claude-pro
Step 3 of 3
Select "3rd-party platform" when prompted

When you see "Select login method", choose option 3. 3rd-party platform (Amazon Bedrock, Microsoft Foundry, or Vertex AI). This tells Claude to use the proxy URL and API key you configured. You only need to do this once - it will remember your choice.

You're done! From now on, use claude-pro to access Claude Opus 4.5 with extended thinking. Your regular claude command (if you have one) remains unchanged.
How this works: We add a PowerShell function claude-pro that sets the API key and proxy URL as environment variables, then launches Claude Code. Your regular claude command stays unchanged.
Step 1 of 3
Open PowerShell

Open Windows Terminal or PowerShell. You may need to run as Administrator if you haven't created a PowerShell profile before.

Step 2 of 3
Add the claude-pro function to your PowerShell profile

This creates a PowerShell function that sets your API key and proxy URL, then launches Claude Code. Copy and paste this entire block.

if (!(Test-Path $PROFILE)) { New-Item -ItemType File -Force -Path $PROFILE }
Add-Content $PROFILE "`nfunction claude-pro { `$env:ANTHROPIC_AUTH_TOKEN = 'YOUR_API_KEY'; `$env:ANTHROPIC_BASE_URL = 'https://proxy.classicgroup.asia'; claude }"
. $PROFILE
Step 3 of 4
Launch Claude Pro

Run this command to start Claude Code.

claude-pro
Step 4 of 4
Select "3rd-party platform" when prompted

When you see "Select login method", choose option 3. 3rd-party platform (Amazon Bedrock, Microsoft Foundry, or Vertex AI). This tells Claude to use the proxy URL and API key you configured. You only need to do this once.

You're done! From now on, use claude-pro in PowerShell to access Claude Opus 4.5 with extended thinking. Your regular claude command (if you have one) remains unchanged.

Troubleshooting

"Quota exhausted" or rate limit errors
What's happening: The AI model has a limit on how many requests can be made per minute/hour. When you hit this limit, you'll see this error temporarily.
What to do: Simply retry your request. The system automatically rotates between multiple accounts, so retrying usually works immediately. If it persists, wait 30-60 seconds and try again.
Connection timeout or "could not connect"
What's happening: Your computer couldn't reach the proxy server. This is usually a network issue on your end.
What to do: Check your internet connection. Try opening https://proxy.classicgroup.asia/health in your browser - you should see a JSON response with "status": "ok". If not, the server may be temporarily down; try again in a few minutes.
"claude-pro: command not found"
What's happening: The alias/function wasn't added to your shell config, or your shell hasn't reloaded the config.
What to do: Close your terminal completely and open a new one. If that doesn't work, re-run Step 1 from the instructions above.
Login prompt or "Select login method" appears
What's happening: The environment variables weren't set correctly before Claude launched.
What to do: Make sure you're running claude-pro (not just claude). If you just added the alias, run source ~/.zshrc (Mac/Linux) or restart your terminal and try again.

Need help? Contact your provider with the exact error message you're seeing.
